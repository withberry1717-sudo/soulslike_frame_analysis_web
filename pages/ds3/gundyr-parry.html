英雄グンダ：パリィ成立〜致命の一撃

英雄グンダの薙ぎ払い攻撃をパリィし、致命の一撃へ移行するまでの流れをフレーム単位で整理する。攻撃の予備動作、接触、パリィVFX、敵リアクション、致命演出、出血VFX、カメラ復帰、操作可能までを分析対象にした。

概要

このページでは、パリィ対象攻撃と致命攻撃を分けて分析する。パリィ対象攻撃は攻撃アニメーション開始と思われる地点を0F、致命攻撃は致命開始を0Fとして記録する。

フレーム定義：パリィ対象攻撃＝攻撃開始を0F / 致命攻撃＝致命開始を0F

1. パリィ対象攻撃：予備動作〜成立

この攻撃は、41Fから62Fまで長く溜めを見せた後、63Fから一気に薙ぎ払いへ移る。78Fで武器が目前に迫り、80Fで接触する。見てから反応するより、予備動作を読んで対応する攻撃になっている。

3. まとめ

英雄グンダのパリィ成立から致命攻撃までの流れは、成功判定、VFX、敵リアクション、カメラ、ダメージUIを細かく分けて見せている。特に、接触80FからパリィVFX85F、敵リアクション92Fへと段階的に進むことで、プレイヤーは「受け止めた」「成功した」「敵が崩れた」を順番に認識できる。

致命攻撃では、33Fから溜め、39Fからカメラ寄り、48〜50Fで刺突、52Fでダメージ表示、56Fから出血VFXと続く。さらに、82Fから両手持ちに移行し、149F以降で剣を引き抜く二段目の報酬を作っている。操作不能時間は長いが、数十Fごとに新しい視覚情報が出るため、退屈になりにくい。

観点分析結果
予備動作41F〜62Fで攻撃を読ませ、63F以降に一気に加速する。
成功演出接触、VFX、敵リアクションを数フレームずらして出す。
報酬演出刺す、抜くの二段構成でパリィ成功の価値を大きく見せる。
カメラヒット前のズーム、沈み込み、ブラー、画面揺れで衝撃を補強する。
操作復帰演出カメラからロックオンカメラへ段階的に戻し、致命開始から293Fで操作可能にする。

4. 自作ゲームへの応用

自作アクションで受け流しやパリィに近いシステムを作る場合、成功した瞬間にすべてを同時に出すのではなく、「接触 → 成功VFX → 敵リアクション → 反撃受付 → ダメージ → よろけ」の順で出すと、プレイヤーが成功を理解しやすい。

  • 敵攻撃接触から3〜5F後に成功VFXを出す
  • さらに数F後に敵の姿勢崩しを入れる
  • 反撃前に短いカメラ寄りやヒットストップを入れる
  • ダメージ表示とVFXを少しずらし、情報を読みやすくする
  • 長い演出を入れる場合は、数十Fごとに新しい変化を作る
